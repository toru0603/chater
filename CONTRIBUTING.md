# CONTRIBUTING

ありがとうございます！このプロジェクトへの貢献を歓迎します。

## 進め方

- 小さな修正（typo、ドキュメント修正など）は直接 PR を送ってください。
- 大きな設計変更や機能追加は先に Issue を作成して議論してください。

## 開発環境

前提: Python 3.10+

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

開発サーバを起動するには:

```bash
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

テストの実行:

```bash
pytest -q
```

## ブランチ命名

- feature/<短い説明>
- fix/<短い説明>
- docs/<短い説明>

## コミットメッセージ

- 1 行の要約（50 文字以内）

- 空行

- 必要なら詳細説明を追加

関連する Issue がある場合は `Fixes #<番号>` を使って参照してください。

## プルリクエストの作成

- ベースブランチ（通常は main）に向けて PR を作成してください。
- テストが通ること、必要なドキュメント更新を含めることを確認してください。

## コードスタイル

- PEP8 準拠を基本とします。自動整形ツール（black など）の使用を推奨します。

## セキュリティ

- シークレット（API キー、パスワード等）は絶対に含めないでください。見つけた場合は速やかに削除し、関連する PR は取り下げてください。
